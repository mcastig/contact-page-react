# Contact Page — React + TypeScript + Vite

A responsive contact/sales-inquiry page built with React 19, TypeScript, and Vite. Features a fully-validated multi-field form with accessible error messaging and a success state.

## Getting started

```bash
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser.

## Commands

| Command | Description |
|---|---|
| `npm run dev` | Start dev server with HMR |
| `npm run build` | Type-check then build for production |
| `npm run lint` | Run ESLint |
| `npm run preview` | Preview production build locally |
| `npm test` | Run unit tests (single pass) |
| `npm run test:watch` | Run tests in watch mode |
| `npm run test:coverage` | Run tests and generate coverage report |

## Testing

Tests are written with [Vitest](https://vitest.dev/) and [React Testing Library](https://testing-library.com/docs/react-testing-library/intro/). The suite covers all components with 100% statement, branch, function, and line coverage.

```bash
npm test               # run once
npm run test:coverage  # run with coverage (outputs to coverage/)
```

Coverage is generated by V8 and written to `coverage/` as HTML and LCOV reports.

## Project structure

```
src/
  components/
    ContactForm/   # Multi-field form with validation logic
    FormField/     # Labeled text input with error display
    Logo/          # Inline SVG logo
    SelectField/   # Labeled select with error display
  pages/
    ContactPage/   # Page layout: header + hero copy + form
  test/
    setup.ts       # jest-dom matchers setup
  App.tsx
  main.tsx
public/
  icons.svg        # SVG icon sprite
```

## Architecture

**Styling:** Plain CSS with CSS custom properties. Global tokens live in `src/index.css`; component styles use CSS Modules (`.module.css`). Dark mode is handled via `@media (prefers-color-scheme: dark)` — no CSS-in-JS or utility framework.

**SVG icons:** Loaded via `<use href="/icons.svg#icon-name">` referencing a sprite at `public/icons.svg`.

**Assets:** Files under `src/assets/` are imported directly into TSX (Vite handles URL transformation). Files under `public/` are served as static assets.

**TypeScript config:** Split across `tsconfig.json` (references), `tsconfig.app.json` (src files), and `tsconfig.node.json` (Vite config). ESLint uses `typescript-eslint` with `eslint-plugin-react-hooks` and `eslint-plugin-react-refresh`.
